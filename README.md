# Seemse AI

<div align="center">

### 企业级AI助手平台

一款**基于 RuoYi-AI 二次开发**的开源分级权限知识库系统，继承若依生态的高可用性、易扩展性，聚焦「分级权限管控 + 智能知识管理」核心场景，解决传统知识库权限混乱、信息泄露、查找困难等痛点，实现知识安全存储与高效共享的平衡。

## 🔗 项目由来

本项目基于 [RuoYi-AI](https://github.com/ageerle/ruoyi-ai)（若依 AI 增强版）开发，保留了若依框架的**权限管理、系统配置、用户管理**等核心能力，同时针对「知识库场景」进行深度定制：



* 扩展「知识分类、知识上传 / 编辑 / 检索」专属模块

* 强化「精细化权限分级、知识安全管控」核心功能

* 集成「AI 语义检索、知识图谱可视化」智能能力

* 适配「企业 / 组织级知识管理」的协同与合规需求

> 感谢 RuoYi-AI 团队提供的优秀基础框架，本项目将持续同步上游框架的稳定性更新与安全补丁。

## 🌟 核心特性（基于 RuoYi-AI 增强）

### 1. 精细化权限管控（核心增强）



* 基于 RuoYi 原有权限体系，扩展「浏览 / 编辑 / 管理 / 超级」四级知识库专属权限

* 支持「角色自定义权限 + 资源级权限隔离」，适配多部门 / 多场景权限需求

* 动态权限调整：临时授权、项目周期权限自动回收、离职员工权限一键冻结

* 权限继承机制：支持按组织架构层级继承权限，简化大规模团队配置

### 2. 全场景知识管理（新增模块）



* 多格式支持：文档（Word/PDF/Markdown）、图片、视频、音频等，兼容 RuoYi-AI 附件存储能力

* 结构化存储：多级分类归档（部门 / 项目 / 主题），关联 RuoYi 组织架构，支持分类权限管控

* 智能检索：集成 RuoYi-AI 的 AI 语义分析能力，支持关键词 / 全文检索 + 模糊匹配 + 分类筛选

* 知识图谱：可视化展示知识间关联关系，助力快速溯源与关联学习

### 3. 安全与合规保障（深度优化）



* 内容安全：继承 RuoYi 数据加密存储，新增知识水印、下载权限管控、敏感词过滤

* 操作审计：基于 RuoYi 日志系统，扩展知识库专属操作日志（访问 / 编辑 / 下载 / 分享），支持追溯与导出

* 合规适配：满足政企单位涉密资料管理需求，支持权限变更审计、敏感知识访问预警

### 4. 团队协同能力（新增功能）



* 多人在线编辑：支持实时协作修改文档，标注意见追溯，关联 RuoYi 用户体系

* 自定义审核流程：配置多节点审核规则，关联 RuoYi 角色权限，确保知识准确性

* 知识互动：支持评论、点赞、收藏、分享，促进团队知识流转

### 5. 继承 RuoYi-AI 核心优势



* 完善的用户 / 角色 / 菜单权限体系，开箱即用

* 前后端分离架构，易于二次开发与扩展

* 支持 Docker 部署，快速搭建环境

* 内置 AI 能力（文本摘要、翻译、OCR 等），可扩展知识智能处理功能

## 📋 适用场景



* 企业内部培训与知识库建设（权限隔离 + 知识沉淀）

* 跨部门 / 分公司知识共享（分级授权 + 组织架构适配）

## 🚀 快速开始

### 环境要求（与 RuoYi-AI 保持一致）



* 后端：Java 1.8+

* 前端：Vue 3 + Element Plus

* 数据库：MySQL 8.0+ / PostgreSQL 13+

* 中间件：Redis 6.0+（缓存）、MinIO（文件存储，可选）

* JDK：1.8+

* Maven：3.6+

* Node：14+

### 安装部署

#### 1. 克隆仓库



```
\# 克隆本项目

git clone https://github.com/seemse/seemse_ai.git

cd rbkbs

\# （可选）查看 RuoYi-AI 上游代码

git remote add upstream https://github.com/ageerle/ruoyi-ai.git
```

#### 2. 配置环境（基于 RuoYi-AI 配置扩展）



* 复制配置文件模板：



```
cp .env.example .env
```



* 编辑 `.env` 文件，配置数据库、Redis、MinIO 等参数（与 RuoYi-AI 配置兼容）：



```
\# 数据库配置（同 RuoYi-AI）

DB\_HOST=localhost

DB\_PORT=3306

DB\_NAME=rbkbs

DB\_USER=root

DB\_PASSWORD=your-password

\# Redis 配置（同 RuoYi-AI）

REDIS\_HOST=localhost

REDIS\_PORT=6379

REDIS\_PASSWORD=your-redis-password

\# 服务配置

PORT=8080

LOG\_LEVEL=info

\# 知识库专属配置

\# 知识存储路径（本地存储时使用）

KNOWLEDGE\_STORAGE\_PATH=./uploads/knowledge

\# 水印配置（可选）

WATERMARK\_ENABLE=true

WATERMARK\_CONTENT="RBKBS 知识库 - 禁止外泄"

\# AI 功能配置（继承 RuoYi-AI，可选启用）

AI\_ENABLE=true

AI\_API\_KEY=your-ai-api-key（如 OpenAI/本地化模型密钥）
```

#### 3. 初始化数据库


* 导入初始数据：`sql/seemse_ai.sql`（初始角色、权限、测试用户、默认分类）

#### 4. 启动服务（与 RuoYi-AI 启动方式一致）

##### 后端启动（Spring Boot）



```
cd backend

mvn clean package -Dmaven.test.skip=true

java -jar target/ruoyi-ai-rbkbs.jar
```

##### 前端启动（Vue 3）



```
cd frontend

npm install

npm run dev
```

#### 5. 访问系统



* 前端地址：`http://localhost:8081`（默认端口，与 RuoYi-AI 一致）

* 初始账号（继承 RuoYi-AI 并扩展知识库角色）：


* 超级管理员（含知识库超级权限）：`admin` / `admin123`

* 知识库管理员（仅知识库管理权限）：`admin` / `admin123`

* 普通用户（仅浏览 / 编辑授权知识）：`user` / `user123`

## 📖 使用指南

### 1. 核心操作流程（基于 RuoYi-AI 扩展）

#### （1）角色与权限配置（继承 RuoYi 权限体系）



1. 登录超级管理员账号，进入「系统管理 > 角色管理」

2. 新建 / 编辑角色（如「部门知识库管理员」），勾选「知识库权限」模块下的对应权限（浏览 / 编辑 / 管理）

3. 进入「用户管理」，为用户分配角色与所属部门

#### （2）知识库管理（新增功能）



1. 登录知识库管理员账号，进入「知识库 > 分类管理」，创建多级分类（如「技术文档 > 后端开发 > Java」）

2. 进入「知识库 > 知识管理」，上传 / 编辑知识内容，选择分类、访问权限范围（公开 / 仅本部门 / 指定角色）

3. 提交后触发审核流程（可在「系统管理 > 流程管理」配置审核节点），审核通过后对外共享

#### （3）知识检索与使用



* 首页搜索框输入关键词，支持全文检索、分类筛选、AI 语义匹配

* 普通用户可查看授权范围内的知识，编辑者可修改内容，管理员可批量管理与权限调整

## 🔄 与 RuoYi-AI 的差异对比



| 功能点     | RuoYi-AI（上游）     | RBKBS（本项目）        |
| ------- | ---------------- | ----------------- |
| 核心定位    | 通用 AI 增强型后台管理系统  | 专注分级权限知识库管理       |
| 知识库模块   | 无                | 新增（分类、知识、检索、协作）   |
