🔐 分级权限知识库系统功能结构

本系统采用三层架构设计，实现安全可控、灵活扩展、权限精细的企业级知识库管理。
🏗️ 一、核心支撑层 —— 系统底层能力保障
云端大模型模块
提供知识库问答所需的语义理解与内容生成核心能力
支持对接主流云端大模型：通义千问（Qwen）、Deepseek、豆包（Doubao） 等
可根据业务场景动态切换或组合多个模型，实现最优效果与成本平衡
所有模型调用均通过 MCP（Model Context Protocol） 标准化封装，确保兼容性与可维护性

⚙️ 二、核心功能层 —— 系统核心业务与权限管控
1. 知识库管理模块
   负责知识文档的解析、切片、向量化与存储
   为问答系统提供高质量、结构化的数据基础
   支持 PDF、Word、Excel、TXT 等多种格式自动处理
2. 数据内容管理模块
   打通外部业务系统，拓展知识来源
   支持对接：报价系统、商品系统、CRM、ERP 等
   通过 MCP 接口实现外部数据自动同步至知识库
   支持定时/触发式数据更新，确保知识实时性
3. 后台管理模块（核心管控入口）
   🔒 权限管理（重点安全机制）
   多维权限控制：从人员维度与内容维度双重保障信息安全
   支持按 部门、角色、标签、筛选条件 设置访问策略
   权限粒度精细到：具体用户、具体文档、具体内容片段
   可设定“禁止某人/某部门查询某类敏感信息”
   权限由上传者或授权内容管理者设定
   严防大模型信息泄露：所有问答结果均受权限策略实时过滤，确保“看得到 ≠ 能问到”
   📤 文档上传
   提供统一后台上传入口，支持批量导入与元数据标注
   也允许授权用户在前端自行上传，提升协作效率
   所有上传内容自动继承上传者所属的权限策略

💬 三、用户交互层 —— 用户直接操作与互动入口
1. 用户端
   面向普通用户的操作界面
   提供文本交互入口，支持自然语言查询知识库
   支持用户自主上传文档（需权限校验）
   查询结果严格遵循后台设定的权限规则
2. 语音管理
   集成语音转文本（ASR） 与 文本转语音（TTS） 能力
   实现语音问答交互，适用于会议、客服、移动端等场景
   语音内容同样受权限体系约束，确保安全合规

🛠️ 技术架构
🏗️ 核心框架
后端架构：Spring Boot 3.4 + Spring AI + Langchain4j
数据存储：MySQL 8.0 + Redis + 向量数据库（Milvus / Weaviate / Qdrant）
前端技术：Vue 3 + Vben Admin + Naive UI
安全认证：Sa-Token + JWT 双重保障
🔧 系统组件
文档处理：PDF、Word、Excel 解析，图像智能分析
实时通信：WebSocket 实时通信，SSE 流式响应
系统监控：完善的日志体系、性能监控、服务健康检查
💡 小贴士：本项目基于 ruoyi-ai 项目进行深度改造，在其基础上完善了大模型和 MCP 的集成调用（原 ruoyi-ai 存在缺陷），支持任意数据库、任意 API 的 MCP 调用，真正实现企业级灵活扩展与安全可控。
